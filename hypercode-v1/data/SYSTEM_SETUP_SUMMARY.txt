## ğŸ—„ï¸ DATABASE TABLES (8 Total)

1. ResearchPaper
   - Stores research documents with metadata
   - Fields: title, authors, abstract, source_url, doi, published_date
   - Status tracking: processing_status, confidence_score

2. ResearchAgent
   - Defines 9 specialized agents + 1 controller
   - Fields: name, agent_type, model_name, system_prompt
   - Performance: success_rate, avg_processing_time, last_used

3. ResearchTask
   - Task queue for research processing
   - Status: pending, running, completed, failed
   - Dependencies: supports parent-child task relationships

4. KnowledgeNode
   - Graph entities: people, concepts, tools, institutions
   - Fields: node_type, label, canonical_form, description
   - Properties: JSON for flexible attributes

5. KnowledgeRelationship
   - Connections between entities
   - Types: author_of, uses, extends, contradicts, cites, etc.
   - Confidence & strength scoring

6. ConflictRecord
   - Tracks knowledge contradictions
   - Resolution: unresolved, resolved, merged, deprecated
   - Evidence tracking & resolution methods

7. paper_agents (Junction Table)
   - Many-to-many: ResearchPaper â†” ResearchAgent

8. task_dependencies (Junction Table)
   - Many-to-many: ResearchTask â†” ResearchTask

          source_url="https://example.com/paper.pdf"

      papers = session.query(ResearchPaper).filter_by(
          processing_status="pending"
